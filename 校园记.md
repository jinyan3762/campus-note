# 项目构建指令：校园记 (CampusNote)

我需要你作为高级全栈工程师，帮我从零生成一个名为“校园记”的图形化本地应用软件。请严格遵守以下技术栈、功能需求和 Git 操作规范。

## 1. 技术栈与环境

- **后端**: Python 3.9+，使用 Flask 框架。
- **数据库**: SQLite (本地文件存储，无需安装额外数据库)。
- **前端**: HTML5 + CSS3 + JavaScript (原生，不使用重型框架，以确保纯本地运行)。
- **UI风格**: 现代化、圆润、护眼的马卡龙色系（背景柔白/淡紫/淡蓝），适合学生群体。
- **运行方式**: 启动 `app.py` 后在本地浏览器访问 `http://127.0.0.1:5000`。

## 2. 核心功能模块详细要求

### 模块一：用户系统 (基础)

- 简单的本地注册/登录（基于 SQLite 存储用户名和加密密码）。
- 登录后进入主控面板（Dashboard）。

### 模块二：校园论坛 (BBS)

- 展示所有帖子列表（标题、作者、发布时间）。
- 支持发布新帖子（标题 + 正文），支持删除自己发布的帖子。
- 支持评论功能（在帖子详情页下方添加评论）。

### 模块三：校园记事本 (Notepad)

- 类似便利贴风格。
- 支持增、删、改笔记，笔记按修改时间倒序排列。
- 支持简单的文本搜索功能。

### 模块四：课表查询 (Schedule)

- 提供一个 5x7 的课表网格（周一至周日，1-5节课）。
- 支持点击格子添加课程（输入课程名、教室）。
- 支持编辑和删除已添加的课程。
- 数据持久化存储于 SQLite。

### 模块五：待办事项 + 转盘决策 (核心亮点)

- **普通模式**: 添加/删除待办事项，标记完成/未完成。
- **转盘模式**:
  - 在主界面提供一个彩色转盘（Canvas 绘制）。
  - 点击“随机抽取”，转盘旋转并停留在某个待办事项上，底部文字高亮显示该事项。
  - **关键结合功能**: 每个待办事项旁边有一个“找搭子做”按钮。
  - 点击该按钮，系统进入“搭子匹配”模块，自动筛选当前在线或最近活跃的、且兴趣标签匹配（或匹配度最高）的用户，并展示匹配结果（模拟匹配逻辑，可根据用户的注册时间/随机种子计算匹配度）。

### 模块六：搭子匹配 (Partner)

- 用户个人资料卡（可编辑昵称、兴趣标签：如“学习”、“运动”、“游戏”、“吃饭”）。
- **匹配逻辑**: 当从待办事项点击“找搭子”进入时，优先显示与当前用户兴趣重合度最高的前 3 名用户。
- 支持向匹配到的搭子发送“组队邀请”（模拟站内信，可在数据库记录邀请状态）。
